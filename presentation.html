Public Grievance Classification Project

The Bottleneck

Why we built this system

  • Volume Overload: Civic departments receive thousands of complaints daily.
  • Manual Triage: Humans must read, classify, and forward each grievance, causing massive delays.
  • Lost Emergencies: Critical issues (e.g., severe power outages) are easily lost in the huge queue.
  • The Goal: Fully automate classification and prioritization for Real-Time Routing.

The Neural Core

How the AI processes data

  • Lexical Analysis: The system "reads" exact words using TF-IDF Vectorization.
  • Sentiment Audit: Calculates the emotional polarity (frustration level) of the text.
  • Multi-Model Architecture:
    • Classifier: Predicts Exact Category (e.g., Sanitation, Traffic).
    • Priority Predictor: Calculates Urgency (Low, Medium, High).
  • Tech Stack: Python, FastAPI, Scikit-Learn (Random Forest).
